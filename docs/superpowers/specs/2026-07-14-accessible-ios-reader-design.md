# iOS 无障碍小说阅读器产品与技术设计

日期：2026-07-14

状态：已确认

目标平台：iOS 15 及以上

## 1. 产品目标

开发一款面向全盲与视弱用户的 iOS 小说阅读器。产品不通过堆叠功能竞争，而是把阅读、听书和本地文件管理做到稳定、可预测、无广告。

设计原则是：

> 视觉上现代丰富，结构上稳定克制，语义上对 VoiceOver 极其清晰。

“极简”指核心任务、入口和操作路径精简，不代表页面只能使用纯文字或缺少视觉层次。

### 1.1 目标用户

- 全盲与视弱用户同等优先。
- 书架、阅读和听书必须支持用户在不看屏幕的情况下独立完成。
- 导入、格式精修和复杂文件管理主要由无视力障碍的协助者完成，但仍保持 VoiceOver 基础可用。

### 1.2 不做的功能

- 登录和注册
- 广告和广告 SDK
- 在线书城、推荐、社区和排行榜
- 视频和短视频
- 用户画像和基于阅读记录的推荐
- DRM 绕过

## 2. 视觉与交互基准

视觉设计以 Stitch 项目 `PureVoice Accessible Reader` 为准：

- 项目 ID：`14191187057079968133`
- 项目链接：https://stitch.withgoogle.com/projects/14191187057079968133
- 设备类型：Mobile，设计宽度 390pt
- 设计系统：`High-Contrast Accessibility System`

主要参考页面：

- 书架首页（现代版）：`159d5e9d6c724d4ea7f6b439aadfd6e6`
- 传统纸质阅读页：`f77a1838bf2145d781c2b5ca0c79356b`
- 导入与管理（现代版）：`a06ec5d312ed4edb9c035da5448d95e5`
- 听书模式（现代版）：`e925f504621342678d53045c02387372`

Stitch 提供视觉、布局和语义参考，生产应用使用原生 SwiftUI 实现，不嵌入生成的 HTML。

### 2.1 视觉规则

- 保留封面、字体层级、进度信息、色彩和现代排版。
- 使用白色或近白背景、深色正文和深蓝色主要操作。
- 不使用广告式内容、渐变装饰、玻璃模糊和无意义动画。
- 卡片圆角不超过 8pt，不嵌套卡片。
- 所有触控目标至少 60pt 高。
- 支持 Dynamic Type 放大到系统最大字号，布局允许纵向增长，不截断核心文字。

### 2.2 VoiceOver 策略

- 不维护单独的“盲人版”页面。
- 同一套现代界面通过原生无障碍语义适配全体用户。
- 装饰性图片对 VoiceOver 隐藏。
- 书籍行将书名、作者和进度组合成一个焦点。
- 次要动作使用上下文菜单或 `accessibilityAction`，避免增加主焦点数量。
- 图标按钮必须提供 `accessibilityLabel`、必要的 `accessibilityHint` 和状态值。
- 阅读正文按章节标题和段落组织，支持标题转子导航。
- VoiceOver 与应用 TTS 不重复朗读同一正文内容。

## 3. 信息架构

应用使用三个位置固定的底部入口：

### 3.1 书架

- 继续阅读
- 最近书籍和全部书籍
- 打开视觉阅读页
- 进入听书模式
- 删除、重命名和查看文件信息

### 3.2 导入

- 从“文件”App 导入 TXT、EPUB 和无 DRM MOBI
- 显示导入进度和失败原因
- TXT 编码检测和章节拆分
- MOBI 转换为内部 EPUB
- 发起付费 LLM 格式精修
- V2 增加网站传书

### 3.3 设置

- 字号、字体、行距、主题和翻页方式
- 默认 TTS 声音和语速
- VoiceOver 相关偏好
- 精修点数和恢复购买
- 隐私、云端数据删除和关于

听书不是固定底部标签。用户从书籍或阅读页进入听书页；离开听书页后，通过全局迷你播放条返回。底部入口的名称和位置不会根据页面状态改变。

## 4. 发布阶段

### 4.1 V1 离线核心

- 书架
- 本地 TXT、EPUB、无 DRM MOBI 导入
- 视觉阅读和仿真翻页
- 目录和阅读进度
- Apple 系统离线 TTS
- 语速和声音选择
- 后台播放、锁屏和耳机控制
- 完整核心 VoiceOver 支持
- 基础阅读设置

### 4.2 V1.1 付费精修

- StoreKit 2 按次购买精修点数
- 明确授权后上传云端
- 清理无价值广告文字
- 章节识别和目录重排
- 修改摘要和结果预览
- 原文件恢复

### 4.3 V2 网站传书

- App 内随机传书身份
- 临时传书码
- 网站上传
- App 待接收队列
- 文件领取和自动过期删除

### 4.4 后续增强

- FluidAudio/Kokoro 精品离线声音
- 睡眠定时
- 经需求验证后的其他格式

## 5. 核心用户流程

### 5.1 导入书籍

`导入 -> 选择本地文件 -> 自动解析 -> 确认书名和目录 -> 加入书架`

- TXT、MOBI 的解析和转换在本机完成。
- 导入页保持 VoiceOver 基础可用，不提供冗长的专用语音向导。
- 处理完成或失败时只提供一次必要的状态提示。
- 失败时区分 DRM、损坏、编码异常和空间不足。

### 5.2 继续阅读

`打开 App -> 继续阅读 -> 恢复上次章节和位置`

- 首页将书名、作者、章节和进度组合成一个焦点。
- 打开书籍后，焦点落在当前章节标题。
- 翻页和换章只播报必要信息。

### 5.3 视觉阅读

- 支持仿真翻页和纵向滚动。
- 点击屏幕中央显示或隐藏工具栏。
- 工具栏包含返回、目录、听书和阅读设置。
- VoiceOver 开启时提供明确的上一页、下一页操作，复杂翻页手势不是唯一入口。
- 正文支持章节标题转子导航。

### 5.4 听书

`阅读页 -> 听书 -> 从当前位置开始朗读`

- 主控制固定为上一段、播放/暂停、下一段。
- 语速和声音使用原生可调整控件。
- 当前朗读文字高亮，视觉页面自动跟随。
- 支持锁屏、耳机和控制中心。
- 电话、导航或其他音频中断后保留播放位置。

### 5.5 LLM 精修

`选择书籍 -> 查看预计点数 -> 同意上传 -> 云端精修 -> 对比结果 -> 应用或放弃`

- 上传前显示字数、费用、处理内容和删除时间。
- 原文件默认保留，精修结果作为新版本。
- 用户确认后才切换阅读版本。
- 云端任务失败不扣点，或自动退回点数。

## 6. 技术架构

### 6.1 技术栈

- SwiftUI：书架、导入、听书、设置和业务界面
- Readium Swift Toolkit 3.8：EPUB 解析、Navigator、定位、目录和 TTS 编排
- UIKit Bridge：在 SwiftUI 中承载 Readium Navigator
- Core Data：iOS 15 本地元数据与状态
- Keychain：传书身份和服务令牌
- StoreKit 2：精修点数购买和恢复
- AVFoundation：Apple 系统 TTS 和音频会话

### 6.2 模块边界

- `LibraryService`：书籍元数据、版本、文件位置和删除
- `ImportService`：格式识别、解析、转换和任务状态
- `ReadingService`：阅读进度、目录和定位
- `SpeechService`：朗读队列、声音、语速和后台控制
- `RefinementService`：上传、任务轮询、结果下载和版本切换
- `PurchaseService`：点数、交易验证和幂等扣除

SwiftUI 页面只通过这些服务访问业务数据，不直接操作解析器、文件系统或支付接口。

### 6.3 统一书籍模型

- EPUB 直接交给 Readium。
- TXT 检测 UTF-8、GB18030、GBK、UTF-16 等编码，识别章节后生成内部 EPUB。
- MOBI 仅处理无 DRM 文件，提取正文、目录、封面和图片后生成内部 EPUB。
- 原始文件始终保留，内部 EPUB 用于阅读和 TTS。
- `libmobi` 仅作为候选方案；因 LGPL-3.0 和 iOS 分发要求，集成前必须完成许可证审查。

### 6.4 本地数据

Core Data 保存：

- 书籍元数据和文件位置
- 原始版与精修版关系
- 当前章节、Locator 和阅读进度
- 显示设置
- TTS 声音、语速和播放位置
- 导入和精修任务状态

### 6.5 TTS

V1 使用 Readium `PublicationSpeechSynthesizer` 与 Apple Speech Synthesis：

- 按句切分
- 播放、暂停、上一句和下一句
- 当前句高亮
- 自动翻页
- 后台播放和远程控制

`SpeechService` 对上层暴露与引擎无关的接口，后续可接入 FluidAudio/Kokoro。

### 6.6 云端服务

精修服务数据流：

1. App 创建带唯一任务号的精修任务。
2. 服务端验证购买和点数状态。
3. App 经 HTTPS 上传用户明确选择的文件。
4. 服务端拆分内容并调用 LLM。
5. 服务端生成结构化结果和修改摘要。
6. App 下载结果并请求用户确认。
7. 服务端按保留策略删除文件。

网站传书与支付身份完全分离，仅通过临时传书码建立短期关联。

## 7. 可靠性与错误处理

- 导入状态：等待、解析、转换、完成、失败。
- 失败不删除原文件，可重新尝试。
- 不支持的 DRM MOBI 明确提示文件受保护。
- 网络、LLM 和支付失败不影响离线阅读。
- 精修任务号保证重试幂等，避免重复扣点。
- 只有可下载结果生成成功后才正式扣点。
- App 被终止后能够恢复导入、精修和听书状态。

## 8. 隐私与安全

- 不读取 IMEI、UDID、IDFA，不生成设备指纹。
- 首次使用传书功能时随机生成 App 专属身份并存入 Keychain。
- 用户可见名称为“传书编号”与“临时传书码”。
- 小说默认只保存在本机。
- LLM 精修每次都需要用户明确授权。
- 云端精修源文件和结果在任务完成后 24 小时自动删除。
- 网站待传文件在 72 小时未领取时自动删除。
- App 提供删除云端任务记录和重置传书身份入口。
- 不向广告或分析服务共享阅读记录和设备标识。

## 9. 验收标准

### 9.1 核心 VoiceOver 任务

以下任务必须能在不看屏幕的情况下独立完成：

1. 打开最近阅读的书。
2. 查找并打开其他书籍。
3. 浏览目录并跳转章节。
4. 上一页、下一页和返回书架。
5. 开始、暂停和恢复听书。
6. 调节语速并切换声音。
7. 从锁屏或耳机控制播放。
8. 删除或重命名一本书。

导入、目录精修和网站传书达到 VoiceOver 基础可用，不作为最高优先级流程。

### 9.2 兼容性与质量

- 覆盖 iOS 15 至当前发布版本。
- 使用真机 VoiceOver 测试，不只依赖 Accessibility Inspector。
- 测试最大 Dynamic Type、粗体文字、增强对比度和减少动态效果。
- 测试中文系统声音、蓝牙耳机、锁屏、电话中断和后台播放。
- 测试超长 TXT、目录异常 EPUB、传统 MOBI/KF8 和损坏文件。
- 连续听书数小时无明显内存增长、进度丢失或重复朗读。
- 使用 TestFlight 邀请真实盲人和视弱用户测试。

## 10. 关键决策记录

- 全盲和视弱用户同等优先。
- iOS 15 为最低系统版本。
- V1 使用 Apple 系统离线 TTS，预留自定义引擎。
- 支持 TXT、EPUB、无 DRM MOBI。
- LLM 精修采用逐次授权的云端处理。
- 精修使用 StoreKit 按次购买点数，不使用订阅。
- 网站传书在 V2 上线。
- 设备身份采用随机 App 标识和临时传书码，不使用硬件标识。
- Stitch 现代版页面为视觉基准；极简约束功能与操作路径，不压缩视觉质量。
